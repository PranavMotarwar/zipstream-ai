# Contributing to zipstream-ai

Thank you for your interest in contributing to **zipstream-ai**.  
We welcome contributions from the community and appreciate your time and effort.

This document outlines how to contribute to the initiative.

---

## Code of Conduct

All contributors are expected to follow the [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).

By participating in this project, you agree to abide by its terms and help maintain a respectful and inclusive environment.

---

## How to Contribute

### 1. Reporting Issues and Bugs

If you encounter a bug, unexpected behavior, or have a feature request:

1. Go to the **Issues** tab on GitHub  
   https://github.com/PranavMotarwar/zipstream-ai/issues
2. Check if a similar issue already exists.
3. If not, open a new issue and include:
   - A clear description of the problem
   - Steps to reproduce (if applicable)
   - Expected vs actual behavior
   - Python version and operating system
   - Sample archive or file structure (if relevant)

---

### 2. Proposing New Features

Feature requests are welcome, especially those that improve:
- Streaming performance
- File format support
- LLM integration
- CLI usability
- Documentation and examples

Please open an issue first to discuss the proposal before starting implementation.
